    1. Determine which city endpoints overlap using `CityEndpoint.endpointsForRegion(lat:lon:span:)` with a default span of 0.04 (covers ~4km)
    2. If no endpoints found, set `errorMessage = "Crime data not available for this area"`, clear incidents, set stats to score=0/label="No Data"/incidentCount=0. Do NOT call loadMockData. Return early.
    3. For each matching endpoint, build the SODA query URL:
       - Base: `endpoint.baseURL`
       - `$where=within_circle(\(endpoint.fieldMapping.geoColumn),\(lat),\(lon),3000) AND \(endpoint.fieldMapping.datetime) > '\(ninetyDaysAgoISO)'`
       - `$limit=5000` (per CRIME-03)
       - `$$app_token=\(Self.appToken)` only if appToken is non-empty (per CRIME-04)

    4. Check `ResponseCache.shared.get(key:layer:)` first — cache key uses `ResponseCache.cacheKey(layer: .crime, lat: lat, lon: lon)`
    5. For cache miss, fire parallel `URLSession.shared.dataTask` requests for each endpoint
    6. Use a `DispatchGroup` to wait for all requests to complete, then merge all incident arrays

    - `parseSFIncidents(from json: [[String: Any]]) -> [CrimeIncident]`: Extract `latitude` (String->Double), `longitude` (String->Double), `incident_category`, `incident_description`, `incident_date`. Guard `lat.isFinite, lon.isFinite`.
    - `parseOaklandIncidents(from json: [[String: Any]]) -> [CrimeIncident]`: Extract `location_1` as `[String: Any]`, then `coordinates` as `[Double]`. CRITICAL: GeoJSON is `[longitude, latitude]` — `coords[0]` = lon, `coords[1]` = lat. Extract `crimetype`, `description`, `datetime`. Guard `lat.isFinite, lon.isFinite`.
    - Route to correct parser based on `endpoint.name`

    **Field validation (per CRIME-06):**
    Before parsing each response, call `validateFields(json, required: endpoint.fieldMapping.requiredFields)`. If any field is missing from the first item:

    - Do NOT fall back to `loadMockData()` — leave incidents empty
    - Log via `AppLogger.network.error("...")`

    **After all fetches complete:**
    - Merge all city incident arrays into single `[CrimeIncident]`
    - Build `DensityGrid.build(from: mergedIncidents, region: MKCoordinateRegion(center: CLLocationCoordinate2D(latitude: lat, longitude: lon), span: MKCoordinateSpan(latitudeDelta: 0.08, longitudeDelta: 0.08)))`
    - Compute density-normalized score from DensityGrid: `let peakDensity = Double(densityGrid?.maxCount ?? 0); let normalized = min(peakDensity / 50.0, 1.0); let score = max(20, Int(100.0 - normalized * 80.0))` — this normalizes against a baseline of 50 incidents per grid cell. Higher density = lower score. Per CONTEXT.md Decision 1: replace old formula with density-normalized scoring.
    - Set `recencyLabel = "Based on incidents from last 90 days"` (per CRIME-07)
    - On main thread: update `self.incidents`, `self.stats`, `self.densityGrid`, `self.recencyLabel`, clear `self.isLoading`
