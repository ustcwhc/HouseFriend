<objective>
Migrate the crime heatmap from CPU pixel-by-pixel CrimeTileOverlay to Mapbox's GPU-accelerated HeatmapLayer, and migrate all annotation types (schools, superfund, earthquake, housing, crime clusters, user pin) from MKAnnotation to Mapbox PointAnnotation.

Purpose: The crime heatmap is the app's signature visual (gas/glow on dark tiles). Moving from CPU tile rendering to GPU HeatmapLayer is a major performance improvement. Annotations are simpler — Mapbox PointAnnotation handles the same use cases with less code.
Output: Crime heatmap glows on dark map, all pin types render and are tappable.
</objective>

class CrimeTileOverlay: MKTileOverlay {
    struct Hotspot {
        let lat: Double
        let lon: Double
        let weight: Double  // 0.0-1.0
    }
    var hotspots: [Hotspot]

    // Glow colors: transparent -> yellow -> orange -> red -> bright red
    static func glowRGB(_ v: Double) -> (UInt8, UInt8, UInt8) {
        if v >= 0.75 { return (255,  30,  30) }
        if v >= 0.55 { return (255,  60,  15) }
        if v >= 0.40 { return (255, 100,  10) }
        if v >= 0.25 { return (255, 150,  30) }
        if v >= 0.12 { return (255, 190,  50) }
        return               (255, 220, 100)
    }
}

    - Create `GeoJSONSource` with FeatureCollection from `crimeHotspots` array
    - Each Feature is a Point at `(hotspot.lat, hotspot.lon)` with property `"weight": hotspot.weight`

    **HeatmapLayer:**
    - Layer ID: `"crime-heat"`
    - Source: `"crime-incidents"`
    - `heatmapWeight`: expression reading the "weight" property:
      ```swift
      .expression(Exp(.get) { "weight" })
      ```
    - `heatmapIntensity`: interpolate by zoom for visibility at different scales:
      ```swift
      .expression(Exp(.interpolate) { Exp(.linear); Exp(.zoom); 0; 1; 9; 3 })
      ```
    - `heatmapRadius`: interpolate by zoom (small dots at low zoom, bigger at high zoom):
      ```swift
      .expression(Exp(.interpolate) { Exp(.linear); Exp(.zoom); 0; 2; 9; 20; 14; 30 })
      ```
    - `heatmapColor`: gradient matching CrimeTileOverlay.glowRGB. Use interpolation expression:
      ```swift
      .expression(
          Exp(.interpolate) {
              Exp(.linear)
              Exp(.heatmapDensity)
              0;   StyleColor(red: 0, green: 0, blue: 0, alpha: 0)           // transparent
              0.12; StyleColor(red: 1.0, green: 0.86, blue: 0.39, alpha: 0.1) // warm yellow faint
              0.25; StyleColor(red: 1.0, green: 0.75, blue: 0.20, alpha: 0.25) // yellow-orange
              0.40; StyleColor(red: 1.0, green: 0.39, blue: 0.04, alpha: 0.4) // orange
              0.55; StyleColor(red: 1.0, green: 0.24, blue: 0.06, alpha: 0.6) // red-orange
              0.75; StyleColor(red: 1.0, green: 0.12, blue: 0.12, alpha: 0.75) // bright red
              1.0;  StyleColor(red: 1.0, green: 0.12, blue: 0.12, alpha: 0.85) // bright red max
          }
      )
      ```
    - `heatmapOpacity`: `.constant(0.85)` for gas-like transparency

    Replace all MKAnnotation-based rendering with Mapbox PointAnnotation in the SwiftUI Map content block. The old `HFAnnotation`, `HFAnnotationData`, and `Coordinator.updateAnnotations` are replaced by declarative Mapbox annotations.

    **Inside the `Map(viewport: $viewport) { }` content block**, add annotation groups using Mapbox's `ForEvery` (or `PointAnnotationGroup`):

    **Schools:**
    ```swift
    if selectedCategory == .schools {
        let levels = ZoomTier(zoom: currentZoom).schoolLevelsToShow()
        PointAnnotationGroup(schools.filter { levels.contains($0.level) }) { school in
            PointAnnotation(coordinate: school.coordinate)
                .image(named: "graduationcap.fill") // Will need to use .iconImage with pre-registered image
                .iconColor(school.level == .elementary ? StyleColor(.systemGreen)
                         : school.level == .middle ? StyleColor(.systemBlue)
                         : StyleColor(.systemPurple))
                .onTapGesture {
                    onSchoolTap(school)
                    return true
                }
        }
    }
    ```

    NOTE: Mapbox PointAnnotation works with pre-registered images, not SF Symbols directly. The approach is:
    1. Register SF Symbol images with the map style in `.onAppear`:
       ```swift
       if let img = UIImage(systemName: "graduationcap.fill") {
           try? mapView.mapboxMap.addImage(img.withTintColor(.white), id: "graduationcap-fill")
       }
       // ... same for other icons
       ```
    2. Then use `.image(named: "graduationcap-fill")` in PointAnnotation.

    **Alternatively**, use `MapViewAnnotation` for SwiftUI-native annotation rendering (simpler for SF Symbols). MapViewAnnotation renders any SwiftUI view at a coordinate:
    ```swift
    if selectedCategory == .schools {
        let levels = ZoomTier(zoom: currentZoom).schoolLevelsToShow()
        ForEvery(schools.filter { levels.contains($0.level) }) { school in
            MapViewAnnotation(coordinate: school.coordinate) {
                Image(systemName: "graduationcap.fill")
                    .foregroundStyle(school.level == .elementary ? .green : school.level == .middle ? .blue : .purple)
                    .font(.title3)
                    .onTapGesture { onSchoolTap(school) }
            }
        }
    }
    ```

    Use `MapViewAnnotation` for all annotation types (it supports SF Symbols natively and SwiftUI views for custom rendering like crime clusters):

    **Superfund sites:**
    - Icon: `exclamationmark.triangle.fill`, color: `.orange`
    - Visible when `selectedCategory == .superfund && ZoomTier(zoom: currentZoom).showsCityAnnotations`
    - Tap: `onSuperfundTap(site)`

    **Earthquake events:**
    - Show magnitude as text (e.g., "4.2"), color by magnitude (>=5 red, >=4 orange, else yellow)
    - Visible when `selectedCategory == .earthquake && ZoomTier(zoom: currentZoom).showsCityAnnotations`
    - No tap callback (earthquakes don't have detail sheets)

    **Supportive housing:**
    - Icon: `house.fill`, color: `.teal`
    - Visible when `selectedCategory == .supportiveHome && ZoomTier(zoom: currentZoom).showsNeighborhoodAnnotations`
    - Tap: `onHousingTap(facility)`

    **User pin (pinned location):**
    - Icon: `mappin`, color: `.red`
    - Visible whenever `pinnedLocation != nil`
    - No tap callback

    **Zoom-based filtering:** Add a `@State private var currentZoom: Double = 14.0` property to track the current zoom level. Update it in the `onCameraChanged` handler. Use it to compute `ZoomTier(zoom: currentZoom)` for visibility filtering.

    **Remove old code:** After all annotations are migrated to MapViewAnnotation:
    - Remove the `HFAnnotationData` enum
    - Remove the `HFAnnotation` class
    - Remove the `annotationMap` dictionary and `updateAnnotations` method (all annotation diffing is now handled by Mapbox)
    - Remove the `dequeue` helper method
    - Remove the `coordinateInsidePolygon` method (ZIP tap is now handled by `onLayerTapGesture` — but keep it for now if population tap still needs it; the layer tap approach from Plan 02 ZIP layer should handle it)

    **ZIP tap via layer gesture:** For the population layer, instead of ray-casting, use Mapbox's built-in feature query. Add `.onLayerTapGesture("zip-fill") { feature, context in ... }` to detect taps on ZIP polygons and extract the zipId from feature properties. This replaces the manual `coordinateInsidePolygon` function.
