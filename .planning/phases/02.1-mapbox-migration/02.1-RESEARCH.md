# Phase 02.1: Mapbox Migration - Research

**Researched:** 2026-03-22
**Domain:** Map SDK migration (Apple MapKit to Mapbox Maps SDK for iOS)
**Confidence:** HIGH

## Summary

This phase replaces Apple MapKit (`MKMapView` via `UIViewRepresentable`) with Mapbox Maps SDK v11 for iOS across all 10 data layers. The current `HFMapView.swift` (748 lines) is the single file being replaced, along with its two custom renderer files (`CrimeTileOverlay.swift`, `NoiseSmokeRenderer.swift`). The Mapbox SDK provides native SwiftUI support via its `Map` view (a `UIViewControllerRepresentable`), which eliminates the need for a hand-rolled `UIViewRepresentable` wrapper. The SDK also provides built-in `HeatmapLayer`, `FillLayer`, `LineLayer`, and `PointAnnotation` types that replace the current `MKTileOverlay`, `MKPolygon`, `MKPolyline`, and `MKAnnotation` usage.

The key motivation is dark map tiles for the crime heatmap (gas/glow effect). Mapbox's Standard style supports a `.night` light preset that provides this out of the box. The current code already forces `map.overrideUserInterfaceStyle = .dark` for the crime layer -- Mapbox replaces this with proper styled dark tiles. Additionally, Mapbox's built-in `HeatmapLayer` can replace the hand-rolled `CrimeTileOverlay` pixel-by-pixel Gaussian renderer, which is a significant simplification.

**Primary recommendation:** Use Mapbox Maps SDK v11 SwiftUI `Map` view as the primary integration path. Use `MapReader` to access underlying `MapView` for advanced features (coordinate bounds calculation, camera state observation). Replace `CrimeTileOverlay` with Mapbox's native `HeatmapLayer` fed by a GeoJSON source. Replace `NoiseSmokeRenderer` with stacked `LineLayer` instances using `lineBlur` and `lineOpacity` for the smoke effect.

### Pattern 1: SwiftUI Map with MapReader
**What:** Use Mapbox's native SwiftUI `Map` view with `MapReader` for advanced UIKit-level access
**When to use:** Primary map integration pattern
**Example:**
```swift
// Source: https://docs.mapbox.com/ios/maps/guides/swift-ui/
import SwiftUI
import MapboxMaps

struct HFMapView: View {
    @Binding var viewport: Viewport

    var body: some View {
        MapReader { proxy in
            Map(viewport: $viewport) {
                // Declarative map content goes here
                // Annotations, layers, etc.
            }
            .mapStyle(.standard(lightPreset: .night))  // Dark tiles for crime
            .onMapTapGesture { context in
                onMapTap(context.coordinate)
            }
            .onMapLongPressGesture { context in
                onMapLongPress(context.coordinate)
            }
            .onAppear {
                // Access underlying MapView via proxy.map for advanced ops
            }
        }
    }
}
```

### Pattern 2: GeoJSON Source + Layer for Overlays
**What:** All overlay data (polygons, polylines) goes through GeoJSON sources with typed layers
**When to use:** Replaces MKPolygon, MKPolyline, MKTileOverlay
**Example:**
```swift
// Source: https://docs.mapbox.com/ios/maps/guides/styles/work-with-layers/
// In the Map content block or via MapReader:
var source = GeoJSONSource(id: "fire-zones")
source.data = .featureCollection(fireZoneFeatureCollection)
try mapView.mapboxMap.addSource(source)

var fillLayer = FillLayer(id: "fire-fill", source: "fire-zones")
fillLayer.fillColor = .expression(/* color by severity */)
fillLayer.fillOpacity = .constant(0.4)
try mapView.mapboxMap.addLayer(fillLayer)
```

### Pattern 3: HeatmapLayer for Crime
**What:** Replace pixel-by-pixel CrimeTileOverlay with GPU-accelerated HeatmapLayer
**When to use:** Crime heatmap rendering
**Example:**
```swift
// Source: https://docs.mapbox.com/ios/maps/examples/heatmap/
var crimeSource = GeoJSONSource(id: "crime-incidents")
crimeSource.data = .featureCollection(crimeFeatureCollection)
try mapView.mapboxMap.addSource(crimeSource)

var heatmap = HeatmapLayer(id: "crime-heat", source: "crime-incidents")
heatmap.heatmapWeight = .expression(
    Exp(.interpolate) { Exp(.linear); Exp(.get) { "weight" }; 0; 0; 1; 1 }
)
heatmap.heatmapColor = .expression(/* red-orange-yellow gradient */)
heatmap.heatmapRadius = .expression(
    Exp(.interpolate) { Exp(.linear); Exp(.zoom); 0; 2; 9; 20 }
)
try mapView.mapboxMap.addLayer(heatmap)
```

### Pattern 4: Stacked LineLayer for Smoke Effect
**What:** Replace NoiseSmokeRenderer with multiple LineLayer instances at different widths/blur/opacity
**When to use:** Noise road smoke/haze effect
**Example:**
```swift
// Outer haze layer
var outerHaze = LineLayer(id: "noise-haze-outer", source: "noise-roads")
outerHaze.lineWidth = .expression(/* baseWidth * 12 */)
outerHaze.lineColor = .constant(StyleColor(.init(white: 0.08, alpha: 1.0)))
outerHaze.lineOpacity = .expression(/* 0.025 * intensity */)
outerHaze.lineBlur = .constant(8.0)
try mapView.mapboxMap.addLayer(outerHaze)

// Mid haze layer
var midHaze = LineLayer(id: "noise-haze-mid", source: "noise-roads")
midHaze.lineWidth = .expression(/* baseWidth * 6 */)
// ... similar pattern for inner smoke and core line
```

### Pattern 5: PointAnnotation for Markers
**What:** Replace MKAnnotation/MKAnnotationView with Mapbox PointAnnotation in SwiftUI
**When to use:** Schools, superfund, earthquake, housing, crime cluster pins
**Example:**
```swift
// Source: https://docs.mapbox.com/ios/maps/guides/swift-ui/
Map(viewport: $viewport) {
    ForEvery(schools) { school in
        PointAnnotation(coordinate: school.coordinate)
            .image(named: "graduationcap-fill")
            .iconColor(school.markerColor)
            .onTapGesture {
                onSchoolTap(school)
                return true
            }
    }
}
```

### Pattern 6: Light/Dark Style Switching
**What:** Switch between light and dark map tiles per-layer
**When to use:** Crime layer needs dark tiles; other layers use default/light
**Example:**
```swift
// Source: https://docs.mapbox.com/ios/maps/guides/styles/set-a-style/
// Dark style for crime layer:
.mapStyle(.standard(lightPreset: .night))

// Light style for other layers:
.mapStyle(.standard(lightPreset: .day))

// Or via MapReader for runtime switching:
proxy.map?.mapboxMap.setStyleImportConfigProperty(
    for: "basemap",
    config: "lightPreset",
    value: selectedCategory == .crime ? "night" : "day"
)
```

### Anti-Patterns to Avoid
- **Hand-rolling UIViewRepresentable for Mapbox MapView:** Use the SDK's built-in SwiftUI `Map` view instead -- it handles the UIViewControllerRepresentable bridge internally
- **Rebuilding all overlays on every SwiftUI update:** Use GeoJSON source partial updates (`addGeoJSONSourceFeatures`, `updateGeoJSONSourceFeatures`) for incremental changes
- **Pixel-by-pixel tile rendering:** Use Mapbox's GPU-accelerated `HeatmapLayer` instead of rendering PNG tiles on CPU
- **One source per feature:** Batch features into FeatureCollections in a single GeoJSON source per layer type
- **Embedding overlay metadata in title strings:** The current code encodes data in `MKOverlay.title` (e.g., `"noise:55:2:r"`). In Mapbox, use GeoJSON feature properties and style expressions instead

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Crime heatmap tiles | CPU pixel-by-pixel Gaussian renderer (CrimeTileOverlay) | Mapbox `HeatmapLayer` with GeoJSON point source | GPU-accelerated, 10x+ faster, built-in color gradient expressions |
| Polygon rendering | MKPolygonRenderer with manual color logic | Mapbox `FillLayer` with data-driven expressions | Expressions handle color-by-property automatically |
| Line rendering with blur | Custom MKOverlayRenderer multi-pass drawing | Stacked `LineLayer` instances with `lineBlur` property | Native GPU blur, no manual CGContext drawing |
| Point-in-polygon hit testing | Ray-casting algorithm (`coordinateInsidePolygon`) | Mapbox `onLayerTapGesture("layer-id")` + `queryRenderedFeatures` | SDK handles hit testing against rendered features |
| Tile coordinate math | Web Mercator tile bounds calculation | Mapbox handles tile management internally | All tile slicing is handled by the vector tile engine |
| Annotation diffing | Manual annotationMap key-based diff | Mapbox annotation manager handles add/remove diffing | SDK diffs annotations internally when data changes |

**Key insight:** The current codebase contains ~400 lines of custom rendering code (CrimeTileOverlay + NoiseSmokeRenderer) and ~200 lines of overlay/annotation management that Mapbox's layer system and annotation APIs handle natively. The migration eliminates this custom code.

## Common Pitfalls

### Pitfall 1: Missing .netrc for SPM Download
**What goes wrong:** SPM fails to resolve Mapbox package with authentication error
**Why it happens:** Mapbox SDK download requires a secret token (with `Downloads:Read` scope) in `~/.netrc`
**How to avoid:** Create `~/.netrc` with `machine api.mapbox.com / login mapbox / password <SECRET_TOKEN>` and `chmod 600 ~/.netrc` before adding the SPM dependency
**Warning signs:** Xcode shows "failed to resolve dependencies" or 401/403 errors during package resolution

### Pitfall 2: Access Token Not Set Before Map Init
**What goes wrong:** Map shows blank/error or crashes with "access token required"
**Why it happens:** `MapboxOptions.accessToken` must be set before any `Map()` view is created
**How to avoid:** Set the token in `HouseFriendApp.init()` or `ContentView.init()` before body is evaluated. Read from `.mapbox_token` file or Info.plist `MBXAccessToken` key
**Warning signs:** Blank map area, console errors about missing token

### Pitfall 3: MKCoordinateRegion vs Mapbox Viewport
**What goes wrong:** Binding mismatches when ContentView still uses `@Binding var region: MKCoordinateRegion`
**Why it happens:** Mapbox uses `Viewport` (camera-based) not `MKCoordinateRegion` (center+span)
**How to avoid:** Replace `MKCoordinateRegion` bindings in ContentView with Mapbox `Viewport`. Use `Viewport.camera(center:zoom:bearing:pitch:)` for programmatic positioning. Convert existing region to viewport: `Viewport.camera(center: region.center, zoom: zoomForSpan(region.span))`
**Warning signs:** Map doesn't respond to programmatic region changes

### Pitfall 4: Forgetting to Remove MapKit Overlay Imports
**What goes wrong:** Build errors or confusion from mixing MapKit and Mapbox types
**Why it happens:** Both frameworks define types with similar names (e.g., both have coordinate types)
**How to avoid:** Remove `import MapKit` from HFMapView.swift and renderers. Keep MapKit only in files that still need it (SearchCompleterService, LocationService)
**Warning signs:** Ambiguous type references, wrong coordinate types

### Pitfall 5: GeoJSON Source Updates Causing Full Redraws
**What goes wrong:** Map stutters when updating data (e.g., crime incidents change)
**Why it happens:** Setting `.data` on a GeoJSON source replaces the entire dataset
**How to avoid:** Use partial update API: `addGeoJSONSourceFeatures`, `updateGeoJSONSourceFeatures`, `removeGeoJSONSourceFeatures`. For initial load, use `.data = .featureCollection(...)` once, then partial updates
**Warning signs:** Frame drops when panning with active data layer

### Pitfall 6: 1700+ Census Tract Polygons Performance
**What goes wrong:** Map lag when rendering all census tract polygons at once
**Why it happens:** Large GeoJSON with many complex polygon features
**How to avoid:** (1) Use a single GeoJSON source with all tracts as a FeatureCollection -- Mapbox vector tiles handle viewport culling automatically. (2) Set `tolerance` on GeoJSONSource for polygon simplification. (3) Unlike MKMapView, Mapbox does NOT need manual viewport-based add/remove -- the tile engine handles it
**Warning signs:** Slow initial load, frame drops at low zoom levels

### Pitfall 7: Camera Change Flooding
**What goes wrong:** Too many camera change callbacks overwhelming ContentView state updates
**Why it happens:** `onCameraChanged` fires every frame during pan/zoom animations
**How to avoid:** Debounce camera callbacks or use `mapView.mapboxMap.onMapIdle.observe` for settled-state updates. Only trigger data fetches on idle, not on every frame
**Warning signs:** UI jank during pan, excessive network requests

## Code Examples

### Complete Map View Skeleton (SwiftUI)
```swift
// Source: https://docs.mapbox.com/ios/maps/guides/swift-ui/
import SwiftUI
import MapboxMaps

struct HFMapView: View {
    @Binding var viewport: Viewport
    let selectedCategory: CategoryType
    let crimeIncidents: [CrimeIncident]
    let noiseRoads: [NoiseRoad]
    let fireZones: [FireHazardZone]
    // ... other layer data

    var onMapTap: (CLLocationCoordinate2D) -> Void
    var onMapLongPress: (CLLocationCoordinate2D) -> Void

    var body: some View {
        MapReader { proxy in
            Map(viewport: $viewport) {
                // Annotations
                annotationContent

                // Puck for user location
                Puck2D(bearing: .heading)
            }
            .mapStyle(mapStyleForCategory)
            .onMapTapGesture { context in
                onMapTap(context.coordinate)
            }
            .onMapLongPressGesture { context in
                onMapLongPress(context.coordinate)
            }
            .onAppear {
                setupLayers(proxy: proxy)
            }
            .onChange(of: selectedCategory) { _, newCategory in
                updateLayers(proxy: proxy, category: newCategory)
            }
        }
    }

    private var mapStyleForCategory: MapStyle {
        if selectedCategory == .crime {
            return .standard(lightPreset: .night)
        }
        return .standard(lightPreset: .day)
    }
}
```

### GeoJSON FeatureCollection from App Data
```swift
// Converting existing model arrays to GeoJSON for Mapbox sources
import MapboxMaps

extension FireHazardZone {
    var geoJSONFeature: Feature {
        var feature = Feature(geometry: .polygon(Polygon([coordinates.map {
            LocationCoordinate2D(latitude: $0.latitude, longitude: $0.longitude)
        }])))
        feature.properties = ["severity": .string(severity)]
        return feature
    }
}

func fireZoneFeatureCollection(_ zones: [FireHazardZone]) -> FeatureCollection {
    FeatureCollection(features: zones.map { $0.geoJSONFeature })
}
```

### Camera State to Region Conversion
```swift
// For backward compatibility with code that uses MKCoordinateRegion
import MapboxMaps
import MapKit

extension MapboxMap {
    func currentRegion() -> MKCoordinateRegion {
        let bounds = coordinateBounds(for: CGRect(origin: .zero, size: /* map size */))
        let center = cameraState.center
        let span = MKCoordinateSpan(
            latitudeDelta: bounds.northeast.latitude - bounds.southwest.latitude,
            longitudeDelta: bounds.northeast.longitude - bounds.southwest.longitude
        )
        return MKCoordinateRegion(center: center, span: span)
    }
}
```

## Open Questions

1. **Noise smoke effect fidelity**
   - What we know: Mapbox `lineBlur` applies a Gaussian blur to line edges. Multiple stacked `LineLayer` instances can approximate the 4-layer smoke effect.
   - What's unclear: Whether `lineBlur` produces visually identical results to the current custom `NoiseSmokeRenderer` multi-pass CGContext drawing. The current renderer uses 4 layers at widths 12x, 6x, 3x, 1x with different opacities.
   - Recommendation: Implement the stacked LineLayer approach first; if the visual doesn't match, fall back to a single `LineLayer` with adequate `lineBlur` -- the smoke effect is aesthetic, not data-critical.

2. **Crime cluster annotations**
   - What we know: Mapbox `PointAnnotation` supports clustering natively. The current code renders custom cluster circles with count labels.
   - What's unclear: Whether Mapbox's built-in clustering matches the current custom appearance (red/orange/gray circles with white center and count text).
   - Recommendation: Use Mapbox's native `clusterOptions` on the PointAnnotation group first. If visual matching is needed, use `MapViewAnnotation` (SwiftUI view annotations) for custom cluster rendering.

3. **ZIP polygon label annotations**
   - What we know: Current code creates custom `MKAnnotationView` with `UILabel` subview for ZIP code text. Mapbox has `MapViewAnnotation` for SwiftUI view overlays.
   - What's unclear: Performance of 445 `MapViewAnnotation` instances vs using a `SymbolLayer` with text.
   - Recommendation: Use `SymbolLayer` with `textField` expression for ZIP labels -- it's GPU-rendered and handles hundreds of labels efficiently.
